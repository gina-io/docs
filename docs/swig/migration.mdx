# Migrating from Jinja2 or Django

Swig is **Jinja2/Django-inspired**, not drop-in compatible with either. Marker shapes, filter syntax, inheritance, and most tag names line up ŌĆö but concrete constructs do break on paste, especially `is`-tests from Jinja2 and `forloop.*` from Django. This page lists every gap we are aware of, with workarounds.

## What ports cleanly (both directions)

```mermaid
flowchart LR
  subgraph compat["Ports cleanly"]
    A["{{ var }} / {{ var.attr }} / {{ var[key] }}"]
    B["{% if %} / {% elif %} / {% else %} / {% endif %}"]
    C["{% for x in xs %} / {% endfor %}"]
    D["{% block name %} / {% extends 'parent' %}"]
    E["{% include 'f' %}"]
    F["{# comment #}"]
    G["Whitespace control: {%- -%} and {{- -}}"]
    H["Overlapping filters (upper, lower, length, first, last, join, replace, sort, date, safe, escape, striptags, title, capitalize)"]
  end
```

The markers `{{ }}`, `{% %}`, `{# #}` match all three engines. Filter chaining with `|` works the same way. Comments cannot be nested in any of them. Template inheritance is single-parent in all three.

**Surprise:** Swig **does** support Jinja2-style whitespace control (`{%- ... -%}`, `{{- ... -}}`, `{#- ... -#}`) ŌĆö see [Template Syntax ┬¦ Whitespace control](./syntax#whitespace-control). This was not always the case historically; if a reference says otherwise, it is stale.

## From Jinja2 ŌåÆ Swig

### Parse-time blockers

These fail with `Unexpected tag` or `Unable to parse ŌĆ”` at compile time.

| Jinja2 | Swig equivalent | Notes |
| --- | --- | --- |
| `{% if x is defined %}` | `{% if x !== undefined %}` or `{% if x %}` | Swig has **no `is` operator** and no tests. |
| `{% if x is none %}` | `{% if x === null %}` | Same ŌĆö no `is`. |
| `{% if x is iterable %}` | Custom filter or pre-compute in locals | Register a filter: `swig.setFilter('isIterable', fn)`. |
| `{% if x not in xs %}` | `{% if not (x in xs) %}` | Swig's comparator is `in` only ŌĆö `not in` is not a single token. |
| `{{ a // b }}` | `{{ (a / b)\|int }}` (define `int` filter) or `{{ Math.floor(a / b) }}` | Integer division is not lexed. Math object is reachable as a global. |
| `{{ a ** b }}` | `{{ Math.pow(a, b) }}` | Power operator is not lexed. |
| `{% set x %}body{% endset %}` | `{% set x = "body" %}` | Swig only supports the one-liner form. |
| `{% from "f.html" import macro1, macro2 %}` | `{% import "f.html" as ns %}` then `{{ ns.macro1() }}` | Swig has no `from ... import`. |
| `{% with x = 1 %}...{% endwith %}` | `{% set x = 1 %}` (no scoping) | Swig has no `with` tag. `set` assigns to the shared context. |
| `{{ super() }}` | `{% parent %}` | Swig uses a dedicated tag, not a function call. |

### Macros

Swig macros are simpler than Jinja2's.

```swig
{% macro input(name, type) %}
  <input name="{{ name }}" type="{{ type }}">
{% endmacro %}
```

| Jinja2 feature | Swig support |
| --- | --- |
| Positional args | Yes |
| **Default values** (`{% macro foo(x=1) %}`) | **No.** Macro args are bare identifiers. Handle defaults inside the body: `{% if x %}{{ x }}{% else %}1{% endif %}`. |
| `**kwargs` / `varargs` | No. |
| `caller()` block passing | No. |
| Nested `{% macro %}` inside `{% macro %}` | Untested; avoid. |

### Missing filters

Swig ships **26 filters** vs. Jinja2's ~50. The following Jinja2 filters are absent ŌĆö register them via [`swig.setFilter`](./extending#custom-filters) or inline with an `{% if %}`:

```text
abs  attr  batch  center  default  dictsort  filesizeformat  float  forceescape
format  groupby (case mismatch ŌĆö Swig has groupBy)  indent  int  items
list  map  max  min  pprint  random  round  select  selectattr  slice
string  sum  tojson  trim  truncate  urlize  wordcount  wordwrap  xmlattr
```

Common workarounds:

```swig
{# default ŌåÆ if/else #}
{{ x }}{# Jinja2: {{ x|default('N/A') }} #}
{# Swig ŌĆö either register a 'default' filter, or: #}
{% if x %}{{ x }}{% else %}N/A{% endif %}

{# tojson ŌåÆ json #}
{{ data|json }}         {# Swig's name #}
{{ data|tojson }}       {# Jinja2's name ŌĆö does not work #}

{# truncate ŌåÆ register, or use substr + concat #}
{{ s.substr(0, 80) }}ŌĆ”  {# if s is a JS string #}
```

If an application leans heavily on Jinja2 filters, porting the filter library once (via `setFilter`) is cheaper than refactoring every template.

### Missing globals and helpers

Jinja2 gives you `range()`, `dict()`, `lipsum()`, `cycler`, `joiner`, `namespace` out of the box. Swig does not. Pass what you need through `locals`:

```js
swig.renderFile('page.html', {
  range: function (n) { var a = []; for (var i = 0; i < n; i++) a.push(i); return a; }
});
```

```swig
{% for i in range(10) %}ŌĆ”{% endfor %}
```

## From Django ŌåÆ Swig

### Parse-time blockers

| Django | Swig equivalent |
| --- | --- |
| `{{ x\|filter:"arg" }}` | `{{ x\|filter("arg") }}` ŌĆö parens, not colon. |
| `{% if x in xs %}` | Same ŌĆö but note Django's `not in` works; Swig's does not (see above). |
| `{{ block.super }}` | `{% parent %}` |
| `{% with x=1 %}...{% endwith %}` | `{% set x = 1 %}` |
| `{% firstof a b c %}` | `{% if a %}{{ a }}{% elseif b %}{{ b }}{% else %}{{ c }}{% endif %}` |
| `{% regroup %}` | Pre-compute in controller; pass the grouped structure as a local. Swig has a `groupBy` **filter** that covers the simple case. |
| `{% cycle %}` | Use `loop.index` and a modulo: `{{ classes[loop.index0 % classes.length] }}`. |
| `{% now %}` | Pass `now` through locals, or use `{{ new Date()\|date(...) }}`. |
| `{% url %}` / `{% csrf_token %}` / `{% load %}` / `{% ifchanged %}` / `{% lorem %}` / `{% widthratio %}` / `{% debug %}` | Django-only. No equivalent ŌĆö either pre-compute or register a custom tag. |

### `forloop.*` ŌåÆ `loop.*`

Django's loop variables all live under `forloop`; Swig (like Jinja2) uses `loop`.

| Django | Swig |
| --- | --- |
| `forloop.counter` | `loop.index` |
| `forloop.counter0` | `loop.index0` |
| `forloop.revcounter` | `loop.revindex` |
| `forloop.revcounter0` | `loop.revindex0` |
| `forloop.first` | `loop.first` |
| `forloop.last` | `loop.last` |
| `forloop.parentloop` | *Not supported.* Capture into a local before entering the inner loop: `{% set outer = loop %}{% for ŌĆ” %}{{ outer.index }}ŌĆ”{% endfor %}`. |

A project-wide find/replace of `forloop.` ŌåÆ `loop.` handles the common cases; `parentloop` needs case-by-case attention.

### Method calls need parens

Django resolves `x.method` by trying dict lookup, attribute access, then zero-arg method call automatically. Swig does **not** ŌĆö methods must be invoked explicitly.

```swig
{# Django #}
{{ user.get_absolute_url }}

{# Swig #}
{{ user.get_absolute_url() }}
```

This is a common source of silent failures: Django's version returns the URL, Swig's version emits `function() { ... }` (or blank, because functions bypass autoescape). Audit every `.method` access during migration.

### Math in `{{ }}`

Django templates do not allow arithmetic in `{{ }}`; you use `|add:` or compute in the view. Swig allows `+ - * / %` directly:

```swig
{{ price * 0.9 }}          {# valid in Swig, not in Django #}
{{ price|add:"-10" }}      {# Django idiom #}
```

Migration direction matters: **Django ŌåÆ Swig** means Django idioms (`|add:`) break and need to become arithmetic or a Swig filter; **Swig ŌåÆ Django** means arithmetic expressions are illegal and must move into the view.

## Operators

Swig's comparator set is **wider** than either Jinja2 or Django ŌĆö it accepts JavaScript strict equality and aliases for `<`, `>`, etc.

```text
==  !=  ===  !==  <  >  <=  >=  in
and  or  not  &&  ||  !
gt  gte  lt  lte   ŌåÉ Swig-only word aliases for <, <=, >, >=
```

Missing from Jinja2 that Swig also omits: `is`, `is not`, `not in`.
Missing from Jinja2 that Swig **keeps**: strict equality `===`/`!==` (JS-native).

## Autoescape parity

All three engines autoescape `{{ ŌĆ” }}` by default. Opt-out mechanisms differ:

| Engine | Per-value opt-out | Region opt-out |
| --- | --- | --- |
| **Swig** | `{{ x\|safe }}` | `{% autoescape false %}ŌĆ”{% endautoescape %}` |
| **Jinja2** | `{{ x\|safe }}` | `{% autoescape false %}ŌĆ”{% endautoescape %}` |
| **Django** | `{{ x\|safe }}` | `{% autoescape off %}ŌĆ”{% endautoescape %}` (note: `off`, not `false`) |

Swig additionally **bypasses autoescape for function-call output** (e.g. `{{ formatDate(x) }}` is not escaped). Neither Jinja2 nor Django do this. See [Security ŌĆö autoescape](./security#autoescape-is-the-only-default-xss-protection) for why, and the guardrails around it.

## Migration checklist

Use this as a per-template review pass:

1. **Search for `is ` and `is not `** ŌĆö replace each with the closest `===` / `!==` / `!` equivalent, or a custom filter.
2. **Search for `not in`** ŌĆö wrap as `not (x in xs)`.
3. **Search for `forloop.`** (Django only) ŌĆö replace with `loop.`.
4. **Search for `block.super` / `super()`** ŌĆö replace with `{% parent %}`.
5. **Search for `|filter:"arg"`** (Django only) ŌĆö rewrite as `|filter("arg")`.
6. **Search for `{% with %}`** ŌĆö rewrite with `{% set %}` and scope-check.
7. **Search for `{% from "ŌĆ”" import ŌĆ” %}`** ŌĆö rewrite as `{% import "ŌĆ”" as ns %}` + `ns.name`.
8. **Search for `.<method>` without parens** (Django only) ŌĆö add `()` on every method call.
9. **Search for filters** ŌĆö audit against the filter parity table above; register missing ones via [`setFilter`](./extending#custom-filters).
10. **Search for `//` and `**`** ŌĆö rewrite using `Math.floor` / `Math.pow` or a filter.
11. **Run the full test suite** ŌĆö Swig fails loudly at parse time for unknown tags/filters, so most gaps surface on first render rather than silently.
