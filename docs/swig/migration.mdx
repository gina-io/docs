# Migrating from Jinja2, Django, or Twig

Swig shares the **Jinja2 lineage** ‚ÄĒ so Jinja2 itself, Django templates, and Twig (PHP) all look familiar but none are drop-in compatible. Marker shapes, filter syntax, inheritance, and most tag names line up ‚ÄĒ but concrete constructs do break on paste, especially `is`-tests (Jinja2 / Twig), `forloop.*` (Django), and the `~` / `??` / `?:` / `..` / `#{}` operators (Twig). This page lists every gap we are aware of, with workarounds.

:::info Coming in a future release
Swig's next major track ships a pluggable frontend architecture so Twig, Jinja2, and Django templates can render without rewriting. Dedicated frontends will close most of the gaps on this page. Track progress in the [swig roadmap](https://github.com/gina-io/swig/blob/develop/ROADMAP.md).

**Preview available on the `@alpha` channel.** The Twig frontend parser ships as `@rhinostone/swig-twig@2.0.0-alpha.3` alongside `@rhinostone/swig@2.0.0-alpha.3`:

```bash
npm install @rhinostone/swig-twig@alpha  # 2.0.0-alpha.2 is deprecated; this resolves to alpha.3
```

At this stage the package exposes **parser output only** ‚ÄĒ `parser.parse(source, options)` returns swig-core IR you can inspect, but end-to-end rendering (a `render(...)` entry point) is not wired yet and will land in `1.0.0`. The filter catalog includes ~24 Twig-shape ports (strtr-style `replace`, `date`, `number_format`, `batch`, `slice`, `merge`, ‚Ä¶).
:::

## What ports cleanly

Constructs available in every engine. Anything not listed here likely differs ‚ÄĒ check the engine-specific tab below.

| Feature | Canonical syntax | Swig | Jinja2 | Django | Twig |
| --- | --- | :---: | :---: | :---: | :---: |
| Variable output | `{{ var }}` | ‚úÖ | ‚úÖ | ‚úÖ | ‚úÖ |
| Attribute access | `{{ var.attr }}` | ‚úÖ | ‚úÖ | ‚úÖ | ‚úÖ |
| Bracket access | `{{ var[key] }}` | ‚úÖ | ‚úÖ | ‚ÄĒ [^1] | ‚úÖ |
| Filter chain | `{{ x\|upper\|trim }}` | ‚úÖ | ‚úÖ | ‚úÖ | ‚úÖ |
| Conditional | `{% if %} / {% elif %} / {% else %} / {% endif %}` | ‚úÖ | ‚úÖ | ‚úÖ [^2] | ‚úÖ |
| Loop | `{% for x in xs %} / {% endfor %}` | ‚úÖ | ‚úÖ | ‚úÖ | ‚úÖ |
| Inheritance | `{% extends "parent" %}` + `{% block name %}` | ‚úÖ | ‚úÖ | ‚úÖ | ‚úÖ |
| Include | `{% include "f" %}` | ‚úÖ | ‚úÖ | ‚úÖ | ‚úÖ |
| Comment | `{# comment #}` | ‚úÖ | ‚úÖ | ‚ö†ÔłŹ [^3] | ‚úÖ |
| Whitespace trim | `{%- ‚Ä¶ -%}` and `{{- ‚Ä¶ -}}` | ‚úÖ | ‚úÖ | ‚ÄĒ | ‚úÖ |
| Autoescape (default on) | `{{ x }}` is HTML-escaped | ‚úÖ | ‚úÖ | ‚úÖ | ‚úÖ |
| `safe` filter | `{{ x\|safe }}` | ‚úÖ | ‚úÖ | ‚úÖ | ‚úÖ [^4] |
| Overlapping filters | `upper`, `lower`, `length`, `first`, `last`, `join`, `replace`, `sort`, `date`, `escape`/`e`, `striptags`, `title`, `capitalize` | ‚úÖ | mostly | mostly | mostly |

[^1]: Django uses `.key` for both attribute and dict lookup; `[key]` is not valid template syntax.
[^2]: Django added `{% elif %}` in 1.4; earlier versions required nested `{% if %}` / `{% else %}`.
[^3]: Django uses `{# #}` only for single-line comments; multi-line comments need `{% comment %}‚Ä¶{% endcomment %}`. Swig/Jinja2/Twig accept multi-line in `{# #}`.
[^4]: Twig's idiomatic name is `|raw`, but `|safe` is also accepted. Swig accepts both.

Template inheritance is single-parent in all four ‚ÄĒ no mixins, no multiple inheritance. Comments cannot be nested in any of them.

### Parse-time blockers

These fail with `Unexpected tag` or `Unable to parse ‚Ä¶` at compile time.

| Jinja2 | Swig equivalent | Notes |
| --- | --- | --- |
| `{% if x is defined %}` | `{% if x !== undefined %}` or `{% if x %}` | Swig has **no `is` operator** and no tests. |
| `{% if x is none %}` | `{% if x === null %}` | Same ‚ÄĒ no `is`. |
| `{% if x is iterable %}` | Custom filter or pre-compute in locals | Register a filter: `swig.setFilter('isIterable', fn)`. |
| `{% if x not in xs %}` | `{% if not (x in xs) %}` | Swig's comparator is `in` only ‚ÄĒ `not in` is not a single token. |
| `{{ a // b }}` | `{{ (a / b)\|int }}` (define `int` filter) or `{{ Math.floor(a / b) }}` | Integer division is not lexed. Math object is reachable as a global. |
| `{{ a ** b }}` | `{{ Math.pow(a, b) }}` | Power operator is not lexed. |
| `{% set x %}body{% endset %}` | `{% set x = "body" %}` | Swig only supports the one-liner form. |
| `{% from "f.html" import macro1, macro2 %}` | `{% import "f.html" as ns %}` then `{{ ns.macro1() }}` | Swig has no `from ... import`. |
| `{% with x = 1 %}...{% endwith %}` | `{% set x = 1 %}` (no scoping) | Swig has no `with` tag. `set` assigns to the shared context. |
| `{{ super() }}` | `{% parent %}` | Swig uses a dedicated tag, not a function call. |

### Macros

Swig macros are simpler than Jinja2's.

```swig
{% macro input(name, type) %}
  <input name="{{ name }}" type="{{ type }}">
{% endmacro %}
```

| Jinja2 feature | Swig support |
| --- | --- |
| Positional args | Yes |
| **Default values** (`{% macro foo(x=1) %}`) | **No.** Macro args are bare identifiers. Handle defaults inside the body: `{% if x %}{{ x }}{% else %}1{% endif %}`. |
| `**kwargs` / `varargs` | No. |
| `caller()` block passing | No. |
| Nested `{% macro %}` inside `{% macro %}` | Untested; avoid. |

### Missing filters

Swig ships **26 filters** vs. Jinja2's ~50. The following Jinja2 filters are absent ‚ÄĒ register them via [`swig.setFilter`](./extending#custom-filters) or inline with an `{% if %}`:

```text
abs  attr  batch  center  default  dictsort  filesizeformat  float  forceescape
format  groupby (case mismatch ‚ÄĒ Swig has groupBy)  indent  int  items
list  map  max  min  pprint  random  round  select  selectattr  slice
string  sum  tojson  trim  truncate  urlize  wordcount  wordwrap  xmlattr
```

Common workarounds:

```swig
{# default ‚Üí if/else #}
{{ x }}{# Jinja2: {{ x|default('N/A') }} #}
{# Swig ‚ÄĒ either register a 'default' filter, or: #}
{% if x %}{{ x }}{% else %}N/A{% endif %}

{# tojson ‚Üí json #}
{{ data|json }}         {# Swig's name #}
{{ data|tojson }}       {# Jinja2's name ‚ÄĒ does not work #}

{# truncate ‚Üí register, or use substr + concat #}
{{ s.substr(0, 80) }}‚Ä¶  {# if s is a JS string #}
```

If an application leans heavily on Jinja2 filters, porting the filter library once (via `setFilter`) is cheaper than refactoring every template.

### Missing globals and helpers

Jinja2 gives you `range()`, `dict()`, `lipsum()`, `cycler`, `joiner`, `namespace` out of the box. Swig does not. Pass what you need through `locals`:

```js
swig.renderFile('page.html', {
  range: function (n) { var a = []; for (var i = 0; i < n; i++) a.push(i); return a; }
});
```

```swig
{% for i in range(10) %}‚Ä¶{% endfor %}
```

### Parse-time blockers

| Django | Swig equivalent |
| --- | --- |
| `{{ x\|filter:"arg" }}` | `{{ x\|filter("arg") }}` ‚ÄĒ parens, not colon. |
| `{% if x in xs %}` | Same ‚ÄĒ but note Django's `not in` works; Swig's does not (see above). |
| `{{ block.super }}` | `{% parent %}` |
| `{% with x=1 %}...{% endwith %}` | `{% set x = 1 %}` |
| `{% firstof a b c %}` | `{% if a %}{{ a }}{% elseif b %}{{ b }}{% else %}{{ c }}{% endif %}` |
| `{% regroup %}` | Pre-compute in controller; pass the grouped structure as a local. Swig has a `groupBy` **filter** that covers the simple case. |
| `{% cycle %}` | Use `loop.index` and a modulo: `{{ classes[loop.index0 % classes.length] }}`. |
| `{% now %}` | Pass `now` through locals, or use `{{ new Date()\|date(...) }}`. |
| `{% url %}` / `{% csrf_token %}` / `{% load %}` / `{% ifchanged %}` / `{% lorem %}` / `{% widthratio %}` / `{% debug %}` | Django-only. No equivalent ‚ÄĒ either pre-compute or register a custom tag. |

### `forloop.*` ‚Üí `loop.*`

Django's loop variables all live under `forloop`; Swig (like Jinja2) uses `loop`.

| Django | Swig |
| --- | --- |
| `forloop.counter` | `loop.index` |
| `forloop.counter0` | `loop.index0` |
| `forloop.revcounter` | `loop.revindex` |
| `forloop.revcounter0` | `loop.revindex0` |
| `forloop.first` | `loop.first` |
| `forloop.last` | `loop.last` |
| `forloop.parentloop` | *Not supported.* Capture into a local before entering the inner loop: `{% set outer = loop %}{% for ‚Ä¶ %}{{ outer.index }}‚Ä¶{% endfor %}`. |

A project-wide find/replace of `forloop.` ‚Üí `loop.` handles the common cases; `parentloop` needs case-by-case attention.

### Method calls need parens

Django resolves `x.method` by trying dict lookup, attribute access, then zero-arg method call automatically. Swig does **not** ‚ÄĒ methods must be invoked explicitly.

```swig
{# Django #}
{{ user.get_absolute_url }}

{# Swig #}
{{ user.get_absolute_url() }}
```

This is a common source of silent failures: Django's version returns the URL, Swig's version emits `function() { ... }` (or blank, because functions bypass autoescape). Audit every `.method` access during migration.

### Math in `{{ }}`

Django templates do not allow arithmetic in `{{ }}`; you use `|add:` or compute in the view. Swig allows `+ - * / %` directly:

```swig
{{ price * 0.9 }}          {# valid in Swig, not in Django #}
{{ price|add:"-10" }}      {# Django idiom #}
```

Migration direction matters: **Django ‚Üí Swig** means Django idioms (`|add:`) break and need to become arithmetic or a Swig filter; **Swig ‚Üí Django** means arithmetic expressions are illegal and must move into the view.

Twig (PHP) and Swig are **Jinja2 siblings** ‚ÄĒ markers, inheritance, macros, and the core tag set line up. The divergences are mostly at the expression-language level: Twig is PHP-flavoured where Swig is JavaScript-flavoured.

### Parse-time blockers

These fail with `Unexpected tag`, `Unable to parse ‚Ä¶`, or "Mismatched nesting state" at compile time.

| Twig | Swig equivalent | Notes |
| --- | --- | --- |
| `{{ "hello " ~ name }}` | `{{ "hello " + name }}` | Swig has **no `~` operator**. Use `+` (JS concat). If either side may be numeric, cast explicitly ‚ÄĒ JS coerces with `+`. |
| `{{ 1..10 }}` | `range(1, 11)` via locals | No range operator. Pass a `range` helper through locals (see the Jinja2 section). |
| `{{ x ?? 'fallback' }}` | `{% if x %}{{ x }}{% else %}fallback{% endif %}` | No null-coalescing operator. |
| `{{ x ?: 'fallback' }}` | Same as above | Short ternary isn't lexed. |
| `{{ x ? 'a' : 'b' }}` | `{% if x %}a{% else %}b{% endif %}` | No `?:` ternary operator. |
| `{{ x is defined }}` | `{{ x !== undefined }}` or `{{ x }}` | No `is` operator. No tests. |
| `{{ x is null }}` | `{{ x === null }}` | |
| `{{ x is empty }}` | `{{ not x or x.length === 0 }}` or register an `empty` filter | |
| `{{ x is odd }}` / `is even` | `{{ x % 2 === 1 }}` / `{{ x % 2 === 0 }}` | |
| `{{ x is iterable }}` | Custom filter, or pre-compute in locals | |
| `{{ x is divisible by(3) }}` | `{{ x % 3 === 0 }}` | |
| `{{ x is same as(y) }}` | `{{ x === y }}` | |
| `{% set x %}body{% endset %}` | `{% set x = "body" %}` | Swig only supports the one-liner. |
| `{% apply upper %}...{% endapply %}` | `{% filter upper %}...{% endfilter %}` | Swig keeps the pre-Twig-2.9 name. |
| `{% verbatim %}...{% endverbatim %}` | `{% raw %}...{% endraw %}` | |
| `{% embed "layout.twig" %}...{% endembed %}` | `{% extends "layout" %}` in a child template | No `embed` ‚ÄĒ inline the block overrides into a dedicated child. |
| `{% use "blocks.twig" %}` | Not supported | Horizontal reuse has no equivalent. Inline or factor through macros. |
| `{% do x.flush() %}` | `{{ x.flush() }}` or compute in controller | No `do` tag. A function call in `{{ }}` whose return is `""` (or gets escaped away) behaves similarly, but prefer moving side effects out of templates. |
| `{% sandbox %}` | Not supported | No sandbox mode ‚ÄĒ the security model is documented in [Security](./security). |
| `{% deprecated "‚Ä¶" %}` | Not supported | |
| `{{ parent() }}` | `{% parent %}` | Swig uses a dedicated tag, not a function call. |
| `{{ block('name') }}` | Not supported directly | Restructure via inheritance, or pre-compute in the controller. |
| `{{ attribute(obj, key) }}` | `{{ obj[key] }}` | JS-style dynamic access already works; `attribute()` is not provided. |
| `{{ constant('App::VERSION') }}` | Pass through locals | PHP-only concept. |
| `{{ dump(x) }}` | `{{ x\|json }}` | No `dump` helper; `json` filter is the closest. |
| `{% from "f.twig" import m1, m2 %}` | `{% import "f" as ns %}` then `ns.m1()` / `ns.m2()` | Swig has no `from ‚Ä¶ import`. |

### String interpolation

Twig lexes `#{ ‚Ä¶ }` inside double-quoted strings:

```twig
{{ "Hello #{name}!" }}
```

Swig does not ‚ÄĒ concatenate explicitly:

```swig
{{ "Hello " + name + "!" }}
```

Escaping rules are identical on both sides; autoescape still applies to the concatenated output.

### Include with context

```twig
{# Twig #}
{% include 'sidebar.twig' with { user: current } only %}
```

Swig's `{% include %}` accepts a `with` expression (a context object) but has **no `only` modifier** ‚ÄĒ the parent context is always visible unless a variable is shadowed. If strict isolation matters, wrap the included body in a `{% macro %}` and call it: macro bodies only see their arguments plus globals.

```swig
{% include "sidebar" with { user: current } %}
```

### Macros ‚ÄĒ defaults and named args

Twig macros accept default values and named arguments at the call site:

```twig
{% macro input(name, type = 'text', value = '') %}...{% endmacro %}
{{ _self.input(name='email', type='email') }}
```

Swig supports neither. Macro args are bare positional identifiers; defaults happen inside the body:

```swig
{% macro input(name, type, value) %}
  <input name="{{ name }}"
         type="{% if type %}{{ type }}{% else %}text{% endif %}"
         value="{{ value }}">
{% endmacro %}
{{ input("email", "email", "") }}
```

Named-argument call syntax (`input(name="x")`) is not parsed. Imports use the `as ns` form only ‚ÄĒ there is no `{% from ‚Ä¶ import name %}`.

### Filter differences worth flagging

Name / semantic mismatches that bite on paste:

| Twig filter | Swig equivalent |
| --- | --- |
| `default('fallback')` | Not built in ‚ÄĒ use `{% if %}` or register a filter |
| `number_format(2, '.', ',')` | Not built in ‚ÄĒ register, or format in the controller |
| `format('%s ‚ÄĒ %d', name, n)` | Concatenate with `+`, or register a `format` filter |
| `date('Y-m-d')` | Same API ‚ÄĒ Swig's `date` filter also uses PHP format strings |
| `date_modify('+1 day')` | Not supported ‚ÄĒ compute in the controller |
| `merge({ a: 1 })` | Not built in ‚ÄĒ pass the merged object through locals |
| `slice(0, 3)` | `{{ s.slice(0, 3) }}` ‚ÄĒ strings and arrays are JS natives |
| `split(',')` | `{{ s.split(",") }}` ‚ÄĒ call the JS method directly |
| `trim` | `{{ s.trim() }}` |
| `batch(3)` | Not built in ‚ÄĒ group in the controller |
| `keys` | `{{ Object.keys(obj) }}` |
| `raw` | `{{ x\|raw }}` ‚ÄĒ same name, same semantics |
| `escape('html')` / `e` | `{{ x\|e }}` ‚ÄĒ same |
| `url_encode` | `{{ x\|url_encode }}` ‚ÄĒ Swig keeps the underscore (Twig accepts either form) |
| `spaceless` | `{% spaceless %}...{% endspaceless %}` ‚ÄĒ it's a **tag** in Swig, not a filter |

Swig does not distinguish "functions" from "filters" ‚ÄĒ everything is a filter plus whatever you expose via `locals`. Where Twig uses `range(0, 10)`, `max(a, b)`, `min(a, b)`, or `random(xs)`, pass those as locals from the controller.

### `spaceless`

Twig 3 removed the `{% spaceless %}` tag in favour of `{% apply spaceless %}`. Swig keeps the **tag** form (`{% spaceless %}‚Ä¶{% endspaceless %}`) and does not ship a `spaceless` filter. Both sides accept Jinja2-style `{%- ‚Ä¶ -%}` trimming for finer control.

## Operators

Swig's comparator set is **wider** than either Jinja2 or Django ‚ÄĒ it accepts JavaScript strict equality and aliases for `<`, `>`, etc.

```text
==  !=  ===  !==  <  >  <=  >=  in
and  or  not  &&  ||  !
gt  gte  lt  lte   ‚Üź Swig-only word aliases for <, <=, >, >=
```

Missing from Jinja2 / Twig that Swig also omits: `is`, `is not`, `not in`.
Missing from Twig that Swig also omits: `~` (string concat), `??` (null coalesce), `?:` (short ternary), `..` (range).
**Kept by Swig** (absent from Jinja2 / Django / Twig): strict equality `===` / `!==` (JS-native).

## Autoescape parity

All three engines autoescape `{{ ‚Ä¶ }}` by default. Opt-out mechanisms differ:

| Engine | Per-value opt-out | Region opt-out |
| --- | --- | --- |
| **Swig** | `{{ x\|safe }}` | `{% autoescape false %}‚Ä¶{% endautoescape %}` |
| **Jinja2** | `{{ x\|safe }}` | `{% autoescape false %}‚Ä¶{% endautoescape %}` |
| **Django** | `{{ x\|safe }}` | `{% autoescape off %}‚Ä¶{% endautoescape %}` (note: `off`, not `false`) |

**Twig** uses `{% autoescape 'html' %}‚Ä¶{% endautoescape %}` (quoted strategy name, not `false`/`off`); per-value opt-out is also `{{ x\|safe }}` (Twig calls this `|raw`, and Swig accepts both).

Swig additionally **bypasses autoescape for function-call output** (e.g. `{{ formatDate(x) }}` is not escaped). Neither Jinja2, Django, nor Twig do this. See [Security ‚ÄĒ autoescape](./security#autoescape-is-the-only-default-xss-protection) for why, and the guardrails around it.

## Migration checklist

Use this as a per-template review pass:

1. **Search for `is ` and `is not `** (Jinja2 / Twig) ‚ÄĒ replace each with the closest `===` / `!==` / `!` equivalent, or a custom filter.
2. **Search for `not in`** ‚ÄĒ wrap as `not (x in xs)`.
3. **Search for `forloop.`** (Django only) ‚ÄĒ replace with `loop.`.
4. **Search for `block.super` / `super()` / `parent()`** ‚ÄĒ replace with `{% parent %}`.
5. **Search for `|filter:"arg"`** (Django only) ‚ÄĒ rewrite as `|filter("arg")`.
6. **Search for `{% with %}`** ‚ÄĒ rewrite with `{% set %}` and scope-check.
7. **Search for `{% from "‚Ä¶" import ‚Ä¶ %}`** (Jinja2 / Twig) ‚ÄĒ rewrite as `{% import "‚Ä¶" as ns %}` + `ns.name`.
8. **Search for `.<method>` without parens** (Django only) ‚ÄĒ add `()` on every method call.
9. **Search for filters** ‚ÄĒ audit against the filter parity table above; register missing ones via [`setFilter`](./extending#custom-filters).
10. **Search for `//` and `**`** (Jinja2 only) ‚ÄĒ rewrite using `Math.floor` / `Math.pow` or a filter.
11. **Search for `~`** (Twig only) ‚ÄĒ rewrite string concatenation as `+`.
12. **Search for `??` and `?:`** (Twig only) ‚ÄĒ rewrite as `{% if %}{% else %}{% endif %}`.
13. **Search for `..` range literals** (Twig only) ‚ÄĒ replace with a `range()` helper passed through locals.
14. **Search for `#{` inside double-quoted strings** (Twig only) ‚ÄĒ rewrite as `+` concatenation.
15. **Search for `{% embed %}` / `{% use %}` / `{% verbatim %}` / `{% do %}` / `{% sandbox %}` / `{% apply %}` / `{% set x %}‚Ä¶{% endset %}`** (Twig only) ‚ÄĒ see the Twig tab for per-tag rewrites.
16. **Search for `attribute()`, `constant()`, `dump()`** (Twig only) ‚ÄĒ rewrite as `obj[key]`, a local, or `|json`.
17. **Run the full test suite** ‚ÄĒ Swig fails loudly at parse time for unknown tags/filters, so most gaps surface on first render rather than silently.
