# Migrating from Jinja2, Django, or Twig

Swig shares the **Jinja2 lineage** ã so Jinja2 itself, Django templates, and Twig (PHP) all look familiar but none are drop-in compatible. Marker shapes, filter syntax, inheritance, and most tag names line up ã but concrete constructs do break on paste, especially `is`-tests (Jinja2 / Twig), `forloop.*` (Django), and the `~` / `??` / `?:` / `..` / `#{}` operators (Twig). This page lists every gap we are aware of, with workarounds.

:::info Coming in a future release
Swig's next major track ships a pluggable frontend architecture so Twig, Jinja2, and Django templates can render without rewriting. Dedicated frontends will close most of the gaps on this page. Track progress in the [swig roadmap](https://github.com/gina-io/swig/blob/develop/ROADMAP.md).

The Twig frontend already ships on the `@alpha` channel ã see the [Twig Frontend](./twig) section for install, scope, and the full parity matrix.
:::

### Parse-time blockers

These fail with `Unexpected tag` or `Unable to parse ãÎ` at compile time.

| Jinja2 | Swig equivalent | Notes |
| --- | --- | --- |
| `{% if x is defined %}` | `{% if x !== undefined %}` or `{% if x %}` | Swig has **no `is` operator** and no tests. |
| `{% if x is none %}` | `{% if x === null %}` | Same ã no `is`. |
| `{% if x is iterable %}` | Custom filter or pre-compute in locals | Register a filter: `swig.setFilter('isIterable', fn)`. |
| `{% if x not in xs %}` | `{% if not (x in xs) %}` | Swig's comparator is `in` only ã `not in` is not a single token. |
| `{{ a // b }}` | `{{ (a / b)\|int }}` (define `int` filter) or `{{ Math.floor(a / b) }}` | Integer division is not lexed. Math object is reachable as a global. |
| `{{ a ** b }}` | `{{ Math.pow(a, b) }}` | Power operator is not lexed. |
| `{% set x %}body{% endset %}` | `{% set x = "body" %}` | Swig only supports the one-liner form. |
| `{% from "f.html" import macro1, macro2 %}` | `{% import "f.html" as ns %}` then `{{ ns.macro1() }}` | Swig has no `from ... import`. |
| `{% with x = 1 %}...{% endwith %}` | `{% set x = 1 %}` (no scoping) | Swig has no `with` tag. `set` assigns to the shared context. |
| `{{ super() }}` | `{% parent %}` | Swig uses a dedicated tag, not a function call. |

### Macros

Swig macros are simpler than Jinja2's.

```swig
{% macro input(name, type) %}
  <input name="{{ name }}" type="{{ type }}">
{% endmacro %}
```

| Jinja2 feature | Swig support |
| --- | --- |
| Positional args | Yes |
| **Default values** (`{% macro foo(x=1) %}`) | **No.** Macro args are bare identifiers. Handle defaults inside the body: `{% if x %}{{ x }}{% else %}1{% endif %}`. |
| `**kwargs` / `varargs` | No. |
| `caller()` block passing | No. |
| Nested `{% macro %}` inside `{% macro %}` | Untested; avoid. |

### Missing filters

Swig ships **26 filters** vs. Jinja2's ~50. The following Jinja2 filters are absent ã register them via [`swig.setFilter`](./extending#custom-filters) or inline with an `{% if %}`:

```text
abs  attr  batch  center  default  dictsort  filesizeformat  float  forceescape
format  groupby (case mismatch ã Swig has groupBy)  indent  int  items
list  map  max  min  pprint  random  round  select  selectattr  slice
string  sum  tojson  trim  truncate  urlize  wordcount  wordwrap  xmlattr
```

Common workarounds:

```swig
{# default ã if/else #}
{{ x }}{# Jinja2: {{ x|default('N/A') }} #}
{# Swig ã either register a 'default' filter, or: #}
{% if x %}{{ x }}{% else %}N/A{% endif %}

{# tojson ã json #}
{{ data|json }}         {# Swig's name #}
{{ data|tojson }}       {# Jinja2's name ã does not work #}

{# truncate ã register, or use substr + concat #}
{{ s.substr(0, 80) }}ãÎ  {# if s is a JS string #}
```

If an application leans heavily on Jinja2 filters, porting the filter library once (via `setFilter`) is cheaper than refactoring every template.

### Missing globals and helpers

Jinja2 gives you `range()`, `dict()`, `lipsum()`, `cycler`, `joiner`, `namespace` out of the box. Swig does not. Pass what you need through `locals`:

```js
swig.renderFile('page.html', {
  range: function (n) { var a = []; for (var i = 0; i < n; i++) a.push(i); return a; }
});
```

```swig
{% for i in range(10) %}ãÎ{% endfor %}
```

### Parse-time blockers

| Django | Swig equivalent |
| --- | --- |
| `{{ x\|filter:"arg" }}` | `{{ x\|filter("arg") }}` ã parens, not colon. |
| `{% if x in xs %}` | Same ã but note Django's `not in` works; Swig's does not (see above). |
| `{{ block.super }}` | `{% parent %}` |
| `{% with x=1 %}...{% endwith %}` | `{% set x = 1 %}` |
| `{% firstof a b c %}` | `{% if a %}{{ a }}{% elseif b %}{{ b }}{% else %}{{ c }}{% endif %}` |
| `{% regroup %}` | Pre-compute in controller; pass the grouped structure as a local. Swig has a `groupBy` **filter** that covers the simple case. |
| `{% cycle %}` | Use `loop.index` and a modulo: `{{ classes[loop.index0 % classes.length] }}`. |
| `{% now %}` | Pass `now` through locals, or use `{{ new Date()\|date(...) }}`. |
| `{% url %}` / `{% csrf_token %}` / `{% load %}` / `{% ifchanged %}` / `{% lorem %}` / `{% widthratio %}` / `{% debug %}` | Django-only. No equivalent ã either pre-compute or register a custom tag. |

### `forloop.*` ã `loop.*`

Django's loop variables all live under `forloop`; Swig (like Jinja2) uses `loop`.

| Django | Swig |
| --- | --- |
| `forloop.counter` | `loop.index` |
| `forloop.counter0` | `loop.index0` |
| `forloop.revcounter` | `loop.revindex` |
| `forloop.revcounter0` | `loop.revindex0` |
| `forloop.first` | `loop.first` |
| `forloop.last` | `loop.last` |
| `forloop.parentloop` | *Not supported.* Capture into a local before entering the inner loop: `{% set outer = loop %}{% for ãÎ %}{{ outer.index }}ãÎ{% endfor %}`. |

A project-wide find/replace of `forloop.` ã `loop.` handles the common cases; `parentloop` needs case-by-case attention.

### Method calls need parens

Django resolves `x.method` by trying dict lookup, attribute access, then zero-arg method call automatically. Swig does **not** ã methods must be invoked explicitly.

```swig
{# Django #}
{{ user.get_absolute_url }}

{# Swig #}
{{ user.get_absolute_url() }}
```

This is a common source of silent failures: Django's version returns the URL, Swig's version emits `function() { ... }` (or blank, because functions bypass autoescape). Audit every `.method` access during migration.

### Math in `{{ }}`

Django templates do not allow arithmetic in `{{ }}`; you use `|add:` or compute in the view. Swig allows `+ - * / %` directly:

```swig
{{ price * 0.9 }}          {# valid in Swig, not in Django #}
{{ price|add:"-10" }}      {# Django idiom #}
```

Migration direction matters: **Django ã Swig** means Django idioms (`|add:`) break and need to become arithmetic or a Swig filter; **Swig ã Django** means arithmetic expressions are illegal and must move into the view.

Twig (PHP) and Swig are **Jinja2 siblings** ã markers, inheritance, macros, and the core tag set line up. The divergences are mostly at the expression-language level: Twig is PHP-flavoured where Swig is JavaScript-flavoured.

### Parse-time blockers

These fail with `Unexpected tag`, `Unable to parse ãÎ`, or "Mismatched nesting state" at compile time.

| Twig | Swig equivalent | Notes |
| --- | --- | --- |

| `{{ x is defined }}` | `{{ x !== undefined }}` or `{{ x }}` | No `is` operator. No tests. |
| `{{ x is null }}` | `{{ x === null }}` | |

| `{{ x is odd }}` / `is even` | `{{ x % 2 === 1 }}` / `{{ x % 2 === 0 }}` | |

| `{{ x is same as(y) }}` | `{{ x === y }}` | |

| `{% verbatim %}...{% endverbatim %}` | `{% raw %}...{% endraw %}` | |

| `{% use "blocks.twig" %}` | Not supported | Horizontal reuse has no equivalent. Inline or factor through macros. |

### String interpolation

Twig lexes `#{ ãÎ }` inside double-quoted strings:

```twig
{{ "Hello #{name}!" }}
```

Swig does not ã concatenate explicitly:

```swig
{{ "Hello " + name + "!" }}
```

Escaping rules are identical on both sides; autoescape still applies to the concatenated output.

### Include with context

```twig
{# Twig #}
{% include 'sidebar.twig' with { user: current } only %}
```

Swig's `{% include %}` accepts a `with` expression (a context object) but has **no `only` modifier** ã the parent context is always visible unless a variable is shadowed. If strict isolation matters, wrap the included body in a `{% macro %}` and call it: macro bodies only see their arguments plus globals.

```swig
{% include "sidebar" with { user: current } %}
```

### Macros ã defaults and named args

Twig macros accept default values and named arguments at the call site:

```twig
{% macro input(name, type = 'text', value = '') %}...{% endmacro %}
{{ _self.input(name='email', type='email') }}
```

Swig supports neither. Macro args are bare positional identifiers; defaults happen inside the body:

```swig
{% macro input(name, type, value) %}
  <input name="{{ name }}"
         type="{% if type %}{{ type }}{% else %}text{% endif %}"
         value="{{ value }}">
{% endmacro %}
{{ input("email", "email", "") }}
```

Named-argument call syntax (`input(name="x")`) is not parsed. Imports use the `as ns` form only ã there is no `{% from ãÎ import name %}`.

### Filter differences worth flagging

Name / semantic mismatches that bite on paste:

| Twig filter | Swig equivalent |
| --- | --- |
| `default('fallback')` | Not built in ã use `{% if %}` or register a filter |
| `number_format(2, '.', ',')` | Not built in ã register, or format in the controller |
| `format('%s ã %d', name, n)` | Concatenate with `+`, or register a `format` filter |
| `date('Y-m-d')` | Same API ã Swig's `date` filter also uses PHP format strings |
| `date_modify('+1 day')` | Not supported ã compute in the controller |
| `merge({ a: 1 })` | Not built in ã pass the merged object through locals |
| `slice(0, 3)` | `{{ s.slice(0, 3) }}` ã strings and arrays are JS natives |
| `split(',')` | `{{ s.split(",") }}` ã call the JS method directly |
| `trim` | `{{ s.trim() }}` |
| `batch(3)` | Not built in ã group in the controller |
| `keys` | `{{ Object.keys(obj) }}` |
| `raw` | `{{ x\|raw }}` ã same name, same semantics |
| `escape('html')` / `e` | `{{ x\|e }}` ã same |
| `url_encode` | `{{ x\|url_encode }}` ã Swig keeps the underscore (Twig accepts either form) |
| `spaceless` | `{% spaceless %}...{% endspaceless %}` ã it's a **tag** in Swig, not a filter |

Swig does not distinguish "functions" from "filters" ã everything is a filter plus whatever you expose via `locals`. Where Twig uses `range(0, 10)`, `max(a, b)`, `min(a, b)`, or `random(xs)`, pass those as locals from the controller.

### `spaceless`

Twig 3 removed the `{% spaceless %}` tag in favour of `{% apply spaceless %}`. Swig keeps the **tag** form (`{% spaceless %}ãÎ{% endspaceless %}`) and does not ship a `spaceless` filter. Both sides accept Jinja2-style `{%- ãÎ -%}` trimming for finer control.

## Operators

Swig's comparator set is **wider** than either Jinja2 or Django ã it accepts JavaScript strict equality and aliases for `<`, `>`, etc.

```text
==  !=  ===  !==  <  >  <=  >=  in
and  or  not  &&  ||  !
gt  gte  lt  lte   ã Swig-only word aliases for <, <=, >, >=
```

Missing from Jinja2 / Twig that Swig also omits: `is`, `is not`, `not in`.
Missing from Twig that Swig also omits: `~` (string concat), `??` (null coalesce), `?:` (short ternary), `..` (range).
**Kept by Swig** (absent from Jinja2 / Django / Twig): strict equality `===` / `!==` (JS-native).

## Autoescape parity

All three engines autoescape `{{ ãÎ }}` by default. Opt-out mechanisms differ:

| Engine | Per-value opt-out | Region opt-out |
| --- | --- | --- |
| **Swig** | `{{ x\|safe }}` | `{% autoescape false %}ãÎ{% endautoescape %}` |
| **Jinja2** | `{{ x\|safe }}` | `{% autoescape false %}ãÎ{% endautoescape %}` |
| **Django** | `{{ x\|safe }}` | `{% autoescape off %}ãÎ{% endautoescape %}` (note: `off`, not `false`) |

**Twig** uses `{% autoescape 'html' %}ãÎ{% endautoescape %}` (quoted strategy name, not `false`/`off`); per-value opt-out is also `{{ x\|safe }}` (Twig calls this `|raw`, and Swig accepts both).

Swig additionally **bypasses autoescape for function-call output** (e.g. `{{ formatDate(x) }}` is not escaped). Neither Jinja2, Django, nor Twig do this. See [Security ã autoescape](./security#autoescape-is-the-only-default-xss-protection) for why, and the guardrails around it.

## Migration checklist

Use this as a per-template review pass:

1. **Search for `is ` and `is not `** (Jinja2 / Twig) ã replace each with the closest `===` / `!==` / `!` equivalent, or a custom filter.
2. **Search for `not in`** ã wrap as `not (x in xs)`.
3. **Search for `forloop.`** (Django only) ã replace with `loop.`.
4. **Search for `block.super` / `super()` / `parent()`** ã replace with `{% parent %}`.
5. **Search for `|filter:"arg"`** (Django only) ã rewrite as `|filter("arg")`.
6. **Search for `{% with %}`** ã rewrite with `{% set %}` and scope-check.
7. **Search for `{% from "ãÎ" import ãÎ %}`** (Jinja2 / Twig) ã rewrite as `{% import "ãÎ" as ns %}` + `ns.name`.
8. **Search for `.<method>` without parens** (Django only) ã add `()` on every method call.
9. **Search for filters** ã audit against the filter parity table above; register missing ones via [`setFilter`](./extending#custom-filters).
10. **Search for `//` and `**`** (Jinja2 only) ã rewrite using `Math.floor` / `Math.pow` or a filter.
11. **Search for `~`** (Twig only) ã rewrite string concatenation as `+`.
12. **Search for `??` and `?:`** (Twig only) ã rewrite as `{% if %}{% else %}{% endif %}`.
13. **Search for `..` range literals** (Twig only) ã replace with a `range()` helper passed through locals.
14. **Search for `#{` inside double-quoted strings** (Twig only) ã rewrite as `+` concatenation.
15. **Search for `{% embed %}` / `{% use %}` / `{% verbatim %}` / `{% do %}` / `{% sandbox %}` / `{% apply %}` / `{% set x %}ãÎ{% endset %}`** (Twig only) ã see the Twig tab for per-tag rewrites.
16. **Search for `attribute()`, `constant()`, `dump()`** (Twig only) ã rewrite as `obj[key]`, a local, or `|json`.
17. **Run the full test suite** ã Swig fails loudly at parse time for unknown tags/filters, so most gaps surface on first render rather than silently.
