## Project structure

```
docs/                    Markdown source — edit these directly
  getting-started/
  guides/
  concepts/
  reference/
  api/                   Auto-generated — do not edit (see below)
  cli/
  globals/
  roadmap.md
  intro.md
  migration.md
src/                     Docusaurus theme overrides and custom React components
static/                  Static assets (images, favicons)
infra/
  cloudflare/            Cloudflare Worker source
scripts/
  generate-api-docs.js   Pulls JSDoc from the gina package and outputs Markdown
```

## Running locally

**Requirements:** Node.js ≥ 20

```bash
npm install
npm start       # http://localhost:3000/docs/ with live reload
npm run build   # production build → build/
npm run serve   # preview the production build locally
```

## Editing the docs

All documentation lives in `docs/`. Markdown and MDX are both supported. The Docusaurus `editUrl` is configured to point to `gina-io/docs/tree/main/` — the **Edit this page** link at the bottom of each page opens the correct file on GitHub.

When a doc change also involves a framework API, CLI command, or config schema change, update both repos in the same PR cycle and cross-reference them.

---

## Auto-generated API reference

Files under `docs/api/` are generated by `scripts/generate-api-docs.js` from JSDoc comments in the framework source (`gina-io/gina`). Do not edit them directly — changes will be overwritten on the next build.

To update API docs:
1. Edit the JSDoc in the framework source
2. Run `npm run build` here — the `prebuild` step re-generates `docs/api/` automatically

## Gina version sync

The version badge in the navbar (`docusaurus.config.js` → `ginaVersion`) is updated automatically by the framework's `post_publish` script on each release. Do not edit it manually.

## Deploy pipeline

```
push to main
    ↓
GitHub Actions: update-alias.yml
    ↓
Docusaurus build (npm run build)
    ↓
Vercel deploy → gina-io-docs.vercel.app (stable alias updated)
    ↓
Cloudflare Worker proxies gina.io/docs* → Vercel
(strips /docs prefix so Docusaurus receives requests at /)
```

The Cloudflare Worker is deployed separately — see below.

---

## GitHub Actions

| Workflow | Trigger | What it does |
| --- | --- | --- |
| `update-alias.yml` | push to `main` | Builds and deploys to Vercel, updates stable alias |
| `deploy-worker.yml` | push to `main` touching `infra/cloudflare/` | Deploys Cloudflare Worker via Wrangler |
| `build-check.yml` | pull requests | Validates the Docusaurus build before merge |
| `token-expiry-reminder.yml` | every Monday 9am UTC | Opens an issue 2 weeks before secret expiry |

## Cloudflare Worker

Source: `infra/cloudflare/`

The worker routes `https://gina.io/docs*` to Vercel, stripping the `/docs` prefix. This lets Docusaurus run at `baseUrl: '/docs/'` on the public URL while Vercel serves it at `/`.

To update the worker, edit the source under `infra/cloudflare/` and push to `main` — `deploy-worker.yml` handles the deploy via Wrangler automatically.
